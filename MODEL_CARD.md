# Model Card

## Model Overview

AI Nutritionist uses a local scikit-learn `MLPRegressor` as a food-ranking component inside a deterministic meal planner. The model helps rank catalog rows for general wellness meal assembly. It is not clinical, does not diagnose, and does not treat or manage health conditions.

## Intended Use

The ranker is intended for local food ordering inside the CLI and Streamlit app. It supports profile-aware meal assembly across Mediterranean/Greek, omnivore, vegetarian, vegan, and keto-style/low-carb modes.

## Out-Of-Scope Use

This model must not be used for diagnosis, treatment, disease management, therapeutic keto planning, allergy safety, pregnancy nutrition, diabetes or epilepsy management, eating-disorder care, medication interactions, or guaranteed weight change.

## Inputs And Features

Training and ranking features are derived from the local food catalog and profile context. They include nutrient fields, serving size, food group, meal tags, vegetarian/vegan/minimally processed flags, BMI category, age, profile goal, meal target, dietary pattern, and goal-focus context.

## Training Data

The ranker trains locally from the merged USDA/FNDDS-derived catalog and curated Mediterranean/Greek extension. Larger catalogs are sampled deterministically for training. Training rows are generated from a small set of hardcoded profile and meal contexts, then labeled with project heuristics.

No patient records, private health data, registered-dietitian labels, clinical outcomes, or user feedback exports are used for training.

## Labels

Labels are weak labels generated by `score_foods`. They reward meal fit, nutrient density, protein/fiber contribution, lower sodium density, lower saturated-fat density, lower total sugar density, and minimally processed signals. These labels encode project rules; they do not prove nutrition adequacy, safety, preference quality, or health outcomes.

## Algorithm Details

The model uses a scikit-learn MLP regressor with hidden layers `(32, 16)`, ReLU activation, Adam optimization, seeded `random_state=42`, and early stopping. The trained pipeline is cached in-process so repeated recommendations are reproducible for the same code, data, and inputs.

## Ranking Behavior

The planner combines the neural score with deterministic heuristic scoring, preference boosts, avoid-term filters, dietary-pattern filters, meal-slot rules, and guardrails for calories, sodium, saturated fat, sugars, protein, fiber, and food-family repetition. The neural ranker does not produce final meal plans by itself.

## Evaluation

The repository includes tests and a 9-profile evaluation matrix:

```bash
python -m ai_nutritionist.evaluation
```

This evaluation checks product behavior and guidance-alignment signals such as protein presence, produce presence, sodium guardrails, saturated-fat guardrails, and internal quality scores. It is not clinical validation and should not be read as evidence of medical accuracy.

## Privacy And Security

The model trains from committed public/sample catalog rows. Streamlit feedback is session-local unless the user exports it as CSV. Exported feedback is user data and should not be committed.

## Limitations And Failure Modes

- BMI and energy estimates are simplified planning inputs.
- USDA/FNDDS data is US-centric and broad food descriptions may not match exact prepared recipes.
- Curated Mediterranean/Greek rows use estimated USDA-style nutrient components.
- Total sugar values are not equivalent to added sugar values.
- Vegan flags are conservative and may exclude ambiguous plant-based foods.
- Keto-style mode is a low-carbohydrate wellness filter, not a therapeutic ketogenic diet.
- Weak labels can encode project bias and may produce odd combinations, so deterministic guardrails and user review remain required.
